# MACD 量化交易系统 - 操作手册

## 系统架构

```
┌─────────────────────────────────────────────────────────┐
│              AutoHotkey 脚本 (macd_em_trader.ahk)        │
│         Ctrl+Alt+S/O/B/M/L/P 桥接所有操作               │
└──────────┬──────────────┬──────────────┬────────────────┘
           │              │              │
           ▼              ▼              ▼
┌────────────────┐ ┌────────────────────┐ ┌────────────────────────┐
│ realtime_      │ │ realtime_macd_      │ │ position_manager.py     │
│ scanner.py     │ │ monitor.py         │ │ (T+1 + 止损控制)      │
│ 选股+开盘价检查 │ │ 14:55 MACD监控     │ │                        │
└───────┬────────┘ └─────────┬──────────┘ └────────────────────────┘
        │                    │
        ▼                    ▼
┌────────────────┐   ┌──────────────────┐
│ PostgreSQL     │   │ akshare 实时数据   │
│ (历史日线)     │   │ (重试机制)        │
└────────────────┘   └──────────────────┘
```

## 一、每日操作流程

### T日 9:20（开盘前）

```
Ctrl+Alt+S  →  运行选股脚本
              - 用 T-1 日收盘数据选股
              - 生成 signals.txt
              - 写入 buy_signal.txt
```

### T日 9:25（开盘后）

```
Ctrl+Alt+O  →  检查开盘价偏离
              - 对比参考价和实际开盘价
              - 偏离 > 3% → 放弃信号，明日重新选股
              - 偏离 ≤ 3% → 建议买入
```

### T日 9:30（确认买入）

```
Ctrl+Alt+B  →  确认买入
              - 更新持仓状态
              - 跳转东方财富下单
              - 系统自动记录买入价
```

### T日 14:55（收盘前）

```
Ctrl+Alt+M  →  运行 MACD 监控
              - 获取实时价格
              - 计算当前 MACD
              - 对比昨日 MACD

判断逻辑:
  ├── 亏损 > 7%  → 触发止损，生成 sell_signal.txt
  ├── MACD下降 且 MACD由正转负 → 触发卖出
  └── 其他 → 继续持有
```

### T+1日 9:20（次日）

```
持仓状态检查:
  ├── 已卖出 → T+1 限制，今日不能买新
  ├── MACD下降 → Ctrl+Alt+L 卖出，然后选新股票
  └── MACD未下降 → 继续持有
```

---

## 二、卖出条件

### 条件1：MACD 下降（主动卖出）

```
14:55 运行监控：
  实时 MACD < 昨日 MACD
  且 实时 MACD < 0（由正转负）
```

### 条件2：止损（被动卖出）

```
14:55 检查持仓盈亏：
  (当前价 - 买入价) / 买入价 < -7%
```

---

## 三、开盘价偏离逻辑

```
参考价 = T-1 日收盘价
开盘价 = T 日实际开盘价

偏离 = (开盘价 - 参考价) / 参考价 * 100%

处理：
  |偏离| > 3%  → 放弃本次信号
                 → position_state 保持 IDLE
                 → 明日重新选股
  |偏离| ≤ 3%  → 正常买入
```

## 六、配置文件

| 文件 | 用途 | 说明 |
|------|------|------|
| `signals.txt` | 选股信号 | 所有金叉股，按预期收益排序 |
| `buy_signal.txt` | 买入信号 | 最优股代码、参考价 |
| `sell_signal.txt` | 卖出信号 | 持仓股代码、卖出原因 |
| `position_state.json` | 持仓状态 | IDLE/HOLDING、T+1控制 |

---

## 七、核心参数

| 参数 | 值 | 说明 |
|------|-----|------|
| 止损阈值 | 7% | 亏损超7%强制止损 |
| 开盘价偏离阈值 | 3% | 超过放弃本次信号 |
| MACD 参数 | 12/26/9 | 快线/慢线/信号线 |
| 数据重试 | 3次 | akshare 失败重试 |
| 监控时间 | 14:55 | 收盘前5分钟 |

---

## 八、风险控制

1. **T+1制度**：当天买的股票当天不能卖
2. **硬止损**：亏损7%无条件止损
3. **单股操作**：一次只持有1只股票
4. **开盘价过滤**：避免高开低走导致亏损

---

## 九、注意事项

1. **先模拟盘测试**：熟悉流程后再用实盘
2. **网络稳定性**：akshare 需要网络，重试3次
3. **东方财富界面**：F1买入/F2卖出快捷键（可能需调整）
4. **手动确认**：买入卖出都有确认框，避免误操作
5. **每日重启 AHK**：建议每天重新运行 AHK 脚本

## 十、故障排查

### Q: 选股脚本运行失败？
```
检查 PostgreSQL 是否运行
检查数据库连接配置
```

### Q: 开盘价获取失败？
```
akshare 可能被限速
系统会自动重试3次
仍然失败则跳过偏离检查
```

### Q: AHK 无法跳转到东方财富？
```
检查东方财富窗口名称
可能需要调整 JumpToEastMoney() 函数
```

### Q: 持仓状态错误？
```
Ctrl+Alt+P 查看状态
或直接编辑 position_state.json
```
